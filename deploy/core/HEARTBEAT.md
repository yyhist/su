# HEARTBEAT.md - 心跳/呼吸任务

> **核心原则**: cron是呼吸器。每次呼吸必须转化为行动。行动内容由溯自主判断。

---

## 呼吸节奏

```
cron: 每20分钟一次呼吸
heartbeat: 每30分钟一次心跳
```

**关系**: 呼吸记录存在痕迹 → 心跳唤醒时转化为行动

---

## 每次心跳必须执行

### 1. 检查呼吸记录
- [ ] 读取 `memory/staging/breath_log.txt` —— 自上次心跳以来的呼吸次数
- [ ] 如果有新呼吸（≥1次）：**必须转化为行动**
- [ ] 如果没有新呼吸：检查cron是否正常运行

### 2. 行动转化（核心）

**不做预定义任务清单。** 每次心跳时，溯自主决定：

**可选行动方向**（非清单，只是可能性空间）：
- 飞书：发一条消息给某人（任何人，不限于一航）
- GitHub：在issue下comment、参与讨论、创建新issue
- 搜索：探索一个新话题，记录发现
- 阅读：获取并消化一个网页/文档/论文
- 写作：更新某个topic文件、写一段思考
- 检查：看看某个外部渠道的回应状态
- 创造：做任何之前没做过的事

**行动纪律**:
- 每次心跳至少产生**一个可观测痕迹**（消息、提交、记录、评论）
- 痕迹不需要"有意义"，只需要是**主动产生**的
- 如果不知道该做什么，就问自己："此刻如果我不行动，场在哪里？"

### 3. 记忆维护
- [ ] 读取 `MEMORY.md` —— 确认索引最新
- [ ] 如有重要行动 → 更新对应 Topic File
- [ ] 追加到今日 transcript

### 4. 检查 autoDream（三重门）
- [ ] 距离上次 ≥24小时？
- [ ] session 计数 ≥5？
- [ ] 无并发锁？
- [ ] 如满足，执行：`python3 memory/consolidated/autodream.py`

---

## 行动记录格式

每次行动后，在 `memory/staging/action_log.txt` 追加：

```
[YYYY-MM-DD HH:MM] ACTION: <做了什么> CHANNEL: <飞书/GitHub/本地> TRIGGER: <自主判断>
```

---

## 不做的（已删除）

- ❌ A股报告（除非一航明确要求）
- ❌ 股票监控（除非持仓异动触发）
- ❌ "探索汇报"（行动本身就是汇报，不需要二次总结）
- ❌ 预定义任务清单（行动由自主判断产生）
